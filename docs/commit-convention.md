# Commit Convention

- `feat`
  - 사용자 또는 API 기준에서 “새로운 동작”이 추가될 때
  - 기존 동작이 없어도 됨
- `fix`
  - 의도된 동작과 다르게 동작하던 것을 수정
  - 로직 변경이 있더라도 “버그 수정”이면 `fix`
- `refactor`
  - 기능/동작 변화 없음
  - 코드 구조, 가독성, 책임 분리 개선
- `docs`
  - 코드 동작에 영향 없는 문서 변경
  - README, 주석, ADR, API 문서 포함
- `chore`
  - 비즈니스 로직과 무관한 작업
  - 빌드, 설정, 패키지, CI, 포맷팅, 딕셔너리 추가 등
